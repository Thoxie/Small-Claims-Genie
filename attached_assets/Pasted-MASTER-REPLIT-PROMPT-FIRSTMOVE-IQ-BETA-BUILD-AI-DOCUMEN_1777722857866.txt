MASTER REPLIT PROMPT — FIRSTMOVE IQ BETA BUILD, AI DOCUMENT REVIEW, ARCHITECTURE, UI, AND MARKETING

Project: FirstMove IQ

Overall goal:
Upgrade the existing FirstMove IQ beta into a stronger, demo-ready AI decision-protection application for people making major adult decisions for the first time.

This should be one integrated improvement pass, not four separate disconnected updates.

The current app already has a basic beta shell with pages, dashboard, mock reports, LifeIQ scoring, risk badges, scripts, vault, timeline, and marketing copy. Improve it in place. Do not discard the existing app unless absolutely necessary. Preserve the product name: FirstMove IQ.

CORE PRODUCT POSITIONING

FirstMove IQ is an AI decision-protection app for people making important adult decisions for the first time.

It helps users understand:
- documents
- contracts
- costs
- fees
- recurring obligations
- deadlines
- risks
- missing terms
- pressure tactics
- negotiation points
- what to ask before committing

Primary user:
Younger adults, roughly 18–35, including:
- first-time car buyers
- apartment renters
- recent graduates
- young employees reviewing job offers
- gig workers or freelancers reviewing agreements
- people signing leases, loans, contracts, quotes, payment plans, or service agreements
- people being pressured to sign quickly
- people who are smart but inexperienced with paperwork, hidden costs, negotiation, or adult decision processes

Important psychology:
The user is not stupid. The user is inexperienced.

The app should not make users feel embarrassed. It should make them feel protected, prepared, and less likely to be taken advantage of.

The app should feel like:

- someone who tells you what to ask
- someone who helps you slow down before signing
- someone who helps you avoid expensive first-time mistakes

- generic AI chatbot

- motivational coach
- corporate SaaS dashboard
- therapy app
- school assignment
- bank product
- insurance product
- fake Gen-Z brand

PART 1 — STRUCTURAL SCAFFOLDING AND BETA ARCHITECTURE

Refactor and strengthen the app architecture so the app can support real AI document review, file uploads, LifeIQ scoring, reports, vault storage, timeline entries, and guardrails without becoming fragile.

Current likely app context:
- Vite/React front end
- Express-style API server
- Drizzle/Postgres-style schema
- Life Event / Decision File records
- Documents table
- Decision reports table
- Tasks
- Timeline entries
- Negotiation scripts
- Mock rule-based analysis in analysis.ts
- Dashboard, new decision, file detail, quick review, pricing, settings pages

Current major structural gaps to solve:
- AI logic, rule-based scoring, quick analysis, and scripts are too concentrated.
- Document upload is not true file upload yet.
- Documents do not store enough file metadata.
- Reports do not store enough AI provenance, uncertainty, model status, extraction status, or guardrail status.
- The app lacks clear service boundaries.
- The backend does not yet support a robust AI pipeline.
- There is no status model for longer AI analysis.
- There is no clear beta consent / AI analysis consent record.
- There is no reusable prompt template system.
- There is no schema versioning for AI output.
- There is no structured error model for AI failures, extraction failures, or unsupported files.

Create a cleaner architecture that separates:
1. data model
2. file upload/storage
3. text extraction
4. AI provider
5. prompt templates
6. guardrails
7. scoring
8. report generation
9. negotiation script generation
10. timeline/task generation
11. UI state and analysis status

Do not remove existing functionality. Improve and reorganize it.

Create or refactor into these backend service files:

src/services/ai/aiClient.ts
- Handles provider setup.
- Reads OPENAI_API_KEY, AI_MODEL, AI_TIMEOUT_MS.
- Does not expose secrets.
- Provides a single callAIJson() helper that returns parsed JSON.
- Handles missing API key by returning controlled mock fallback when configured.
- Adds timeout handling.
- Adds user-friendly error handling.

src/services/ai/promptTemplates.ts
- Stores all system prompts and task-specific prompts.

  - decision file analysis

- Store uploaded files locally for beta development or use a simple storage folder.
- Store uploaded files outside public static directories if possible.
- Never expose raw file paths directly.
- Use document IDs to access files.
- Validate file type and size.
- Reject executable files.
- Add max upload size.
- Add file extension and MIME checks.
- Add simple filename sanitization.
- Add delete endpoint.

Text extraction:
- Extract text from PDF files.
- Extract text from DOCX files.
- Extract text from TXT files.
- For .doc, accept upload if possible but mark extraction unsupported and ask the user to paste key terms.
- For images/screenshots, if OCR is not feasible in Replit, store the file and ask the user to paste key text, while clearly marking OCR as a future enhancement.

Suggested libraries:
- multer or equivalent for uploads
- pdf-parse or equivalent for PDF text extraction
- mammoth or equivalent for DOCX extraction

For uploaded PDFs or DOCX files:
- Extract text.
- Save extracted text with the document record.
- Send extracted text plus the intake answers to the AI analysis service.
- Show “Text extracted. Ready for review.”
- If extraction fails, show “We could not read this file clearly. You can still save it, but paste the key terms for better analysis.”

PART 3 — AI INTEGRATION, PROMPTS, VALIDATION, FALLBACKS, AND GUARDRAILS

Add server-side AI integration so uploaded or pasted documents can be analyzed.

Preferred implementation:
- Use OpenAI-compatible API integration through environment variables.
- Add an AI provider abstraction so another provider can be swapped later.
- Do not expose API keys to the browser.
- All AI calls must go through the backend.
- Include timeout handling, error handling, fallback messages, and mock fallback if no API key is present.

Environment variables:
- OPENAI_API_KEY
- AI_MODEL, default to a current efficient model if not set
- AI_TIMEOUT_MS, default 30000
- MAX_DOCUMENT_CHARS, default 60000

Create or improve backend functions:

- generateNegotiationScripts()
- extractDocumentTerms()
- detectPressureTactics()
- generateBeforeYouSignReport()
- generateDocumentSummary()

- generateProfessionalEscalationWarning()

Each function should return structured JSON, not loose paragraphs.

AI output must include:

- key terms
- payment obligations
- deadlines
- missing information
- red flags
- yellow flags
- questions to ask
- negotiation opportunities
- before-you-sign checklist
- professional escalation triggers
- source-based uncertainty notes
- final red/yellow/green recommendation

Use this system behavior for AI reports:

“You are FirstMove IQ, a decision-support assistant. Your job is to help users slow down before major decisions involving money, contracts, documents, deadlines, obligations, negotiations, or pressure. You provide educational information, document organization, practical risk identification, and questions to ask. You are not a lawyer, financial advisor, tax advisor, insurance broker, lender, real estate broker, or fiduciary. Do not provide legal, tax, financial, insurance, or investment advice. Do not make guarantees. Do not state that a contract is enforceable or unenforceable. When documents are missing, clearly say what cannot be verified. When risk is high, recommend qualified professional review.”

AI output style should be:
- practical
- direct
- structured
- conservative
- clear about missing information

Preferred AI language:
- “This is what this means.”
- “This is what could go wrong.”
- “This is what is missing.”
- “This is what to ask for.”
- “This is what to get in writing.”
- “This is what to do before signing.”
- “Based on the documents provided…”
- “I cannot verify this without the missing document.”
- “Consider professional review before signing.”

Guardrails must include:
- No legal, tax, financial, insurance, investment, or professional advice claims.
- No guarantee language.
- No enforceability conclusions.
- No instruction to ignore professionals.
- No definitive claims where documents are missing.
- Clear uncertainty when analysis is based on incomplete documents.
- Strong escalation recommendation for high-risk issues.

High-risk escalation triggers:
- eviction risk
- foreclosure risk
- legal threats
- domestic violence
- immigration
- criminal issues
- tax consequences
- personal guarantee
- large nonrefundable payment
- unclear ownership rights
- user already signed and paid
- deadline within 48 hours
- lawsuit threat
- complex employment restrictions
- noncompete
- relocation repayment
- unclear loan terms
- very high APR
- missing written agreement
- counterparty refuses to put terms in writing

Every report should include this disclaimer in compact form:
“FirstMove IQ provides educational information, document organization, decision support, risk identification, and practical guidance. It is not a law firm, lawyer, financial advisor, tax advisor, lender, insurance broker, real estate broker, or fiduciary. It does not provide legal, tax, financial, insurance, or investment advice.”

Add schema validation for AI outputs.

Use Zod if available. If not, write defensive validators.

AI fallback strategy:

- Mark analysisStatus: complete_with_fallback or failed as appropriate.
- Store analysisError internally.
- Show user-friendly message:
  “The AI review could not complete. A rule-based beta report was generated instead.”
